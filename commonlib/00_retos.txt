# 🧩 CML1 — Cliente HTTP profesional con `requests`  
**Objetivo:** dominar HTTP real para scraping, automatización y consumo de APIs.

### Enunciado  
Construye un cliente HTTP reutilizable que permita realizar peticiones GET y POST de forma robusta, configurable y segura.

### Requisitos  
- Uso de `requests.Session()` para mantener cookies y headers.  
- Manejo de headers dinámicos (User-Agent, Referer, Authorization).  
- Manejo de proxies (HTTP/SOCKS).  
- Timeouts configurables.  
- Reintentos automáticos usando `tenacity`.  
- Manejo de errores HTTP (4xx, 5xx) con excepciones personalizadas.  
- Carga de configuración desde `.env` usando `python-dotenv`.  
- Bitácora explicando decisiones de diseño.

### Bonus  
- Implementar un método `get_json()` que valide el JSON antes de retornarlo.  
- Implementar un método `download_file()` con barra de progreso simple (sin tqdm).

# 🧩 CML2 — Scraping estructurado con BeautifulSoup / selectolax  
**Objetivo:** extraer datos limpios, robustos y consistentes desde HTML real.

### Enunciado  
Construye un scraper que reciba una URL, descargue el HTML y extraiga información estructurada usando selectores CSS.

### Requisitos  
- Uso de `requests` para obtener el HTML.  
- Uso de `beautifulsoup4` **o** `selectolax` (si lo instalas).  
- Extracción de múltiples elementos (lista de productos, artículos, etc.).  
- Limpieza de datos (strip, normalización, tipos).  
- Manejo de HTML roto o incompleto.  
- Detección de cambios en estructura (raise custom error).  
- Bitácora explicando cómo diseñaste los selectores y por qué.

### Bonus  
- Crear un scraper genérico con métodos `select_one`, `select_all`.  
- Implementar un detector de cambios basado en hashes del DOM.

# 🧩 CML3 — Scraping dinámico con Playwright  
**Objetivo:** automatizar páginas complejas que requieren JavaScript.

### Enunciado  
Construye un scraper dinámico que navegue a una página, espere elementos específicos y extraiga datos renderizados por JS.

### Requisitos  
- Uso de `playwright` en modo headless.  
- Navegación a una URL.  
- Esperar elementos con `page.wait_for_selector()`.  
- Extracción de texto, atributos y listas.  
- Manejo de timeouts y errores de navegación.  
- Captura de pantallas para debugging.  
- Bitácora explicando cuándo usar Playwright vs requests.

### Bonus  
- Interceptar requests/responses para obtener datos JSON internos.  
- Implementar un método `scrape_paginated()` para múltiples páginas.
